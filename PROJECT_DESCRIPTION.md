# Полное описание проекта: Система управления лабораторией

## Обзор проекта

**Система управления лабораторией** - это профессиональное десктопное приложение для управления медицинским учреждением или лабораторией. Приложение разработано специально для операционных систем Windows 10 и Windows 11 с использованием современных технологий Python и Qt.

### Основное назначение

Приложение предназначено для:
- Управления базой данных пациентов
- Отслеживания процесса лечения пациентов
- Ведения статистики и аналитики
- Создания отчетов и экспорта данных
- Обеспечения безопасности данных и разграничения доступа

### Ключевые характеристики

- **Полностью локальное решение**: Работает без подключения к интернету, все данные хранятся на локальном компьютере пользователя
- **Высокая производительность**: Оптимизировано для работы с базой данных более 100 000 пациентов
- **Современный интерфейс**: Интуитивный пользовательский интерфейс с поддержкой светлой и темной тем
- **Мультиязычность**: Поддержка русского и таджикского языков с возможностью переключения
- **Безопасность**: Шифрование паролей, система прав доступа, журнал действий пользователей

## Архитектура приложения

### Технологический стек

**Backend:**
- **Python 3.13+** - основной язык программирования
- **SQLite** - локальная база данных
- **SQLAlchemy** - ORM для работы с базой данных
- **bcrypt** - хеширование паролей

**Frontend:**
- **PySide6 (Qt6)** - фреймворк для графического интерфейса
- **Matplotlib** - визуализация данных и диаграммы
- **Seaborn** - стилизация графиков

**Обработка данных:**
- **Pandas** - обработка и анализ данных
- **OpenPyXL** - работа с Excel файлами
- **ReportLab** - генерация PDF отчетов

## Функциональность приложения

### 1. Управление пользователями и аутентификация

#### Вход в систему
- Форма авторизации с проверкой учетных данных
- Шифрование паролей с использованием bcrypt
- Запоминание последнего пользователя (опционально)

#### Роли пользователей
Система поддерживает три уровня доступа:

**Администратор:**
- Полный доступ ко всем функциям системы
- Управление пользователями (создание, редактирование, удаление)
- Изменение настроек системы
- Управление резервными копиями
- Просмотр всех журналов действий

**Врач:**
- Просмотр и редактирование карточек пациентов
- Изменение статуса лечения
- Просмотр статистики и аналитики
- Создание и экспорт отчетов
- Добавление заметок и прикрепление файлов

**Регистратор:**
- Добавление новых пациентов
- Просмотр информации о пациентах
- Поиск пациентов
- Запись на прием

### 2. Управление пациентами

#### Карточка пациента
Каждая карточка пациента содержит следующую информацию:

**Личная информация:**
- ФИО (полное имя)
- Телефон
- Год рождения
- Пол (мужской/женский)
- Адрес (опционально)

**Медицинская информация:**
- Тип заболевания
- Название заболевания
- Лечащий врач
- Статус лечения
- Дата следующего приема
- Примечания

**Статусы лечения:**
- 🟡 В процессе лечения
- 🟢 Вылечен
- 🔵 Лечение завершено
- 🔴 Не удалось вылечить
- ⚪ Отказался от лечения
- ⚪ Без изменений

#### Операции с пациентами
- **Создание**: Добавление нового пациента с заполнением всех необходимых полей
- **Редактирование**: Изменение любой информации о пациенте
- **Удаление**: Удаление пациента с каскадным удалением связанных данных
- **Просмотр**: Детальный просмотр информации о пациенте

#### История изменений
Система автоматически ведет журнал всех изменений:
- Кто изменил данные
- Что именно было изменено
- Когда было изменено
- Старые и новые значения

### 3. Поиск и фильтрация

#### Поиск пациентов
Мгновенный поиск по следующим полям:
- ФИО пациента
- Телефон
- ID пациента
- Название заболевания
- Тип заболевания

#### Фильтрация
Возможность фильтрации по:
- Статусу лечения
- Типу заболевания
- Году рождения
- Полу

#### Сортировка
Сортировка по любому полю:
- По возрастанию
- По убыванию
- По ID, ФИО, дате регистрации и т.д.

#### Пагинация
Управляемое отображение данных:
- 25, 50, 100 или 200 записей на страницу
- Навигация по страницам
- Отображение общего количества записей

### 4. Управление файлами

#### Прикрепление файлов
К карточке пациента можно прикрепить:
- PDF документы
- Изображения (результаты анализов, снимки)
- Медицинские документы
- Любые другие файлы

#### Ограничения
- Максимальный размер файла: 100 MB
- Автоматическое создание папки для файлов
- Хранение файлов в локальной файловой системе

### 5. Напоминания

#### Создание напоминаний
- Дата напоминания
- Текст напоминания
- Автоматическое отображение предстоящих напоминаний

#### Управление напоминаниями
- Отметка о выполнении
- Просмотр истории напоминаний
- Автоматическая фильтрация по дате

### 6. Статистика и аналитика

#### Общая статистика
- Общее количество пациентов
- Новых пациентов за месяц
- Новых пациентов за год
- Среднее время лечения (в днях)

#### Статистика по заболеваниям
- Распределение по типам заболеваний
- Топ заболеваний по количеству случаев
- Распределение заболеваний по статусам лечения

#### Статистика лечения
- Распределение по статусам лечения (круговая диаграмма)
- Пациенты по врачам (столбчатая диаграмма)
- Среднее время лечения
- Количество завершенных лечений

#### Активность
- График активности за последние 30 дней
- Количество действий по типам
- Самые активные пользователи

#### Визуализация
- Современные диаграммы с градиентными цветами
- Интерактивные графики с навигацией
- Возможность экспорта графиков в PNG
- Адаптивный дизайн диаграмм

### 7. Экспорт и импорт данных

#### Экспорт в Excel
- Выбор фильтров для экспорта
- Автоматическая настройка ширины колонок
- Поддержка русского языка
- Форматирование данных

#### Экспорт в CSV
- Кодировка UTF-8 с BOM (для корректного отображения в Excel)
- Разделители-запятые
- Поддержка русского языка

#### Экспорт в PDF
- Профессиональное форматирование
- Таблицы с данными пациентов
- Дата создания отчета
- Настройка шрифтов и стилей

#### Импорт из Excel
- Проверка обязательных полей
- Автоматическое определение типов данных
- Обновление существующих записей или пропуск
- Обработка ошибок с логированием

#### Импорт из CSV
- Поддержка UTF-8 с BOM
- Автоматическое определение кодировки
- Валидация данных
- Детальные отчеты об ошибках

### 8. Резервное копирование

#### Создание резервных копий
- Полное резервирование базы данных
- Автоматическое добавление временной метки
- Сжатие для экономии места
- Хранение в отдельной папке

#### Восстановление из резервной копии
- Выбор файла резервной копии
- Проверка целостности данных
- Автоматическое восстановление
- Предупреждение о перезаписи данных

#### Управление копиями
- Автоматическое удаление старых копий (настраиваемое количество)
- Просмотр списка доступных копий
- Информация о размере и дате создания
- Ручное удаление копий

### 9. Настройки приложения

#### Темы оформления
- Светлая тема
- Темная тема
- Мгновенное переключение тем

#### Язык интерфейса
- Русский язык

#### Другие настройки

- Настройки резервного копирования
- Логирование действий

#### Автоматическое логирование
Система автоматически записывает:
- Создание пациента
- Редактирование пациента
- Удаление пациента
- Добавление файлов
- Изменение статуса лечения
- Экспорт/импорт данных
- Создание/восстановление резервных копий

#### Просмотр журнала
- Фильтрация по пациенту

- Фильтрация по типу действия
- Сортировка по дате

## Безопасность

### Шифрование
- Пароли хешируются с использованием bcrypt
- Соль добавляется автоматически
- Невозможно восстановить исходный пароль

### Права доступа
- Три уровня доступа с разными правами
- Проверка прав при каждом действии
- Ограничение доступа к чувствительным функциям

### Журнал действий
- Запись всех действий пользователей
- Невозможность удаления журнала
- Использование для аудита

### Локальное хранение
- Все данные хранятся локально
- Нет передачи данных по сети
- Полный контроль над данными

## Расширение функционала

### Добавление новых функций
1. Создать модель в `models.py`
2. Создать сервис в `services/`
3. Создать UI в `ui/`
4. Обновить `main.py`

### Добавление новых отчетов
1. Создать функцию в `services/export_import.py`
2. Добавить кнопку в UI
3. Подключить к сервису

### Добавление новых диаграмм
1. Создать метод в `services/statistics.py`
2. Добавить метод рисования в `ui/statistics_view.py`
3. Обновить UI для отображения

## Известные ограничения

1. **Платформа**: Приложение разработано только для Windows
2. **Пользователи**: Однопользовательский режим (одновременная работа нескольких пользователей не поддерживается)
3. **Размер файлов**: Максимальный размер вложения - 100 MB
4. **База данных**: SQLite не поддерживает сетевой режим
5. **Языки**: Поддерживаются только русский и таджикский языки
